# Stage 1 사전학습 인체모델 가이드

## 문서 목적

이 문서는 Stage 1에서 "마스킹된 전신 이미지"를 입력으로 받아 체형(beta) 정보를 추출할 때,
사전학습 인체모델이 실제로 어떻게 동작하는지와 "직접 학습이 필요한지"를 명확히 설명합니다.

---

## 1) Stage 1 동작 원리 (마스킹 전신 이미지 입력)

Stage 1의 목표는 다음과 같습니다.

- 입력: 마스킹된 전신 이미지 (RGB 형식)
- 출력: SMPL/STAR 계열 체형 파라미터(beta 10개) + confidence

### 마스킹의 정의 및 모델 호환성

"마스킹"이 정확히 무엇을 의미하는지에 따라 모델 호환성이 달라집니다.

**시나리오 1: 배경만 제거 (권장)**
- 배경: 검은색(0,0,0)으로 채움
- 인체 전체(얼굴 포함): 원본 RGB 값 유지
- 결과: 3채널 RGB 이미지
- 모델 호환성: ✓ 완벽 호환, 최고 성능

**시나리오 2: 배경 제거 + 얼굴 마스킹 (프라이버시)**
- 배경: 검은색(0,0,0)
- 신체(팔/다리/몸통): 원본 RGB 값 유지
- 얼굴: 회색(128,128,128) 등으로 처리
- 결과: 3채널 RGB 이미지
- 모델 호환성: ✓ 호환 가능하지만 성능 약간 저하
  - HMR2/PARE/CLIFF는 얼굴을 주요 특징으로 쓰지 않음 (신체 중심)
  - 신체 실루엣이 대부분 유지되므로 체형 추론 가능
  - 다만 목-얼굴 경계 정보 손실로 상반신 비율 추정에 약간의 오차 가능

**추천:**
- 프라이버시 요구가 없으면 시나리오 1 (배경만 제거)
- 프라이버시 필수면 시나리오 2 (얼굴 회색 처리)
- 어느 경우든 후보 모델들은 입력을 받을 수 있습니다.

일반적인 처리 흐름:

1. 입력 검증
- 해상도/블러/인체 포함률 체크
- 머리~발끝 포함 여부 확인

2. 전처리
- 사람 bbox 정렬 및 크롭
- 배경 제거 → 검은색으로 마스킹 (결과: RGB)
- 리사이즈, 정규화(mean/std)

3. 사전학습 인체모델 추론
- backbone(CNN/ViT)이 RGB 이미지에서 feature 추출
- 회귀 헤드가 pose, beta, camera 파라미터 예측
- beta는 체형 정보로 Stage 2-3에 전달

4. 후처리
- beta 범위 클리핑/안정화
- confidence 계산
- 품질 메타데이터 첨부

핵심 포인트:
- 기존 관절-only 경로와 달리, 실루엣/외곽/텍스처 단서가 포함되어 체형 추정 신호가 증가합니다.
- 마스킹된 RGB 입력은 배경 노이즈를 줄여 인체 중심 학습에 유리합니다.
- 단안 RGB 특성상 절대 깊이 모호성은 남아 있으므로, 거리/구도 조건과 품질 필터가 중요합니다.

## 2) 후보 모델별 입력-출력 관점 설명

## HMR2

- 입력: 전신 RGB 이미지
- 출력: SMPL pose, beta, camera
  - **pose**: 72-dim 벡터 (24개 관절의 회전값)
  - **beta**: 10-dim 체형 파라미터
  - **camera**: 카메라 스케일 파라미터
- 관절 관련 정보:
  - 각 관절의 **회전값(관절 각도)** 제공
  - 3D 절대 위치는 직접 제공하지 않음 (SMPL 신체 모델과 pose로부터 계산 가능)
  - 단일 프레임 기반이므로 시간 궤적은 불가
- 특징:
  - baseline 구축이 빠르고 레퍼런스가 많음
  - 현재 프로젝트에서 1차 도입 후보로 적합

## PARE

- 입력: 전신 RGB 이미지
- 출력: SMPL pose, beta, camera (구현체에 따라 추가 출력 가능)
  - 기본 출력은 HMR2와 동일
  - 일부 구현에서는 관절 confidence 또는 부위별 feature map 제공 가능
- 관절 관련 정보:
  - HMR2와 유사하게 pose 파라미터 제공
  - 부분 가림(occlusion) 상황에서 더 견고한 포즈 추정
- 특징:
  - 가림(occlusion) 상황에 상대적으로 강함
  - 의상/가방/부분 가림이 있는 사용자 환경에서 대안으로 유리

## CLIFF

- 입력: RGB + bbox/사람 중심 정보
- 출력: SMPL pose, beta, camera
  - 출력 형식은 HMR2/PARE와 동일
- 관절 관련 정보:
  - pose 파라미터 제공
  - bbox 정보를 활용해 카메라 스케일 추정을 더 안정적으로 수행
  - → 관절 위치 계산 시 더 일관된 스케일 제공 가능
- 특징:
  - 크롭/스케일 변화가 큰 촬영 조건에서 안정화에 도움
  - bbox 품질 관리가 중요

---

### 참고: 관절 정보 활용 방법

세 모델 모두에서 관절 정보를 얻으려면:

1. **관절 회전값**: pose 파라미터에서 직접 추출
2. **관절 3D 위치**: pose + beta + camera를 SMPL 모델에 입력해 계산
3. **관절 신뢰도**: 일부 구현에서 추가 출력으로 제공

요약:
- 세 모델 모두 "이미지 -> SMPL 파라미터(pose + beta + camera)" 추론이 가능하며,
  현재 목표(Stage 1: Image->Beta)와 구조적으로 맞습니다.
- 추가로 pose 파라미터도 얻을 수 있으므로 Stage 2에서 필요한 관절 정보 활용 가능

## 3) 데이터셋은 무엇을 써야 하나?

질문: "우리가 직접 학습시킬 데이터셋이 필요한가?"

답변: "항상 필수는 아니지만, 서비스 품질을 높이려면 보통 필요합니다."

### 선택지 A: 학습 없이 추론-only 사용

방식:
- 공개 사전학습 체크포인트를 그대로 로드
- 우리 입력 전처리만 맞춘 뒤 inference 수행

장점:
- 가장 빠르게 동작 확인 가능
- 개발 초기 리스크/비용 낮음

한계:
- 우리 서비스 도메인(촬영 거리, 의상, 배경)과 다르면 beta 오차 증가 가능
- 품질 상한이 빠르게 나타남

권장 사용 시점:
- 1차 PoC
- 아키텍처 검증 단계

### 선택지 B: 소규모 파인튜닝 (권장)

방식:
- 사전학습 모델 backbone은 대부분 유지
- 회귀 헤드 또는 일부 레이어만 파인튜닝
- 우리 도메인 샘플로 적응

장점:
- 비용 대비 성능 개선 효율이 좋음
- inference-only 대비 도메인 갭 완화

한계:
- 데이터 정합/학습 파이프라인 필요

권장 사용 시점:
- PoC 통과 후 서비스 전 단계

### 선택지 C: 전면 재학습 (고비용)

방식:
- 모델 전체를 우리 데이터 중심으로 다시 학습

장점:
- 충분한 고품질 데이터가 있을 때 최고 성능 가능

한계:
- 데이터/시간/GPU 비용 큼
- 현재 프로젝트 단계에서는 비효율 가능성 높음

권장 사용 시점:
- 장기 고도화, 대규모 자체 데이터 확보 후

## 4) 추천 데이터셋 구성

실무적으로는 "혼합 전략"이 가장 안전합니다.

1. pretrain/참조 데이터
- AGORA: 체형 다양성, 복잡한 장면
- SSP-3D: 체형(shape) 평가에 유리
- (보조) 3DPW: 실환경 포즈 시퀀스

2. 도메인 적응 데이터
- 우리 서비스 촬영 규격(거리/구도/배경)에 맞춘 내부 샘플
- 마스킹 품질/가림/의상 다양성 반영

권장 학습 순서:
1. 사전학습 모델 추론-only baseline 구축
2. 내부 샘플로 회귀 헤드 파인튜닝
3. 필요 시 부분 레이어 언프리즈

---

## 5) "직접 학습해야 하는 모델인가?" 최종 답변

짧은 답:
- 처음부터 직접 학습은 필수 아님
- 하지만 서비스 품질 목표를 맞추려면 "최소 파인튜닝"은 사실상 권장

의사결정 기준:

1. 추론-only 성능이 목표 지표를 만족하면
- 직접 학습 없이 운영 가능

2. 추론-only가 부족하고 도메인 갭이 보이면
- 헤드 파인튜닝부터 시작

3. 그래도 부족하면
- 부분/전체 재학습 검토

즉, 현재 프로젝트에는 다음 경로가 합리적입니다.

- 1단계: HMR2 또는 PARE 추론-only 연결
- 2단계: 내부 데이터로 소규모 파인튜닝
- 3단계: 필요 시 데이터셋 확장 + 추가 학습

## 6) Stage 1 구현 체크리스트 (실행용)

- [ ] 입력 스키마에 masked_fullbody_image 반영
- [ ] 이미지 품질 검사(전신 포함률, 블러, 해상도) 구현
- [ ] 전처리 규격 고정(bbox, resize, normalize)
- [ ] 사전학습 모델 1종 연결(HMR2/PARE/CLIFF)
- [ ] beta + confidence 출력 표준화
- [ ] Stage 2-3 연동 테스트
- [ ] 추론-only baseline 지표 산출
- [ ] 필요 시 파인튜닝 실험(헤드 우선)

## 7) 운영상 주의사항

- 사용자 촬영 가이드를 API 요구사항으로 명시하는 것이 성능 안정화에 매우 중요
- 카메라 거리/구도 메타데이터를 함께 저장하면 에러 분석과 재학습에 유리
- 마스킹 품질이 낮으면 모델보다 전처리에서 먼저 실패함

권장 촬영 조건:
- 거리 2.0m~3.5m
- 정면 자연 기립
- 머리~발끝 포함
- 과도한 하이/로우 앵글 금지

---

## 결론

- Stage 1은 "마스킹 전신 이미지 -> 사전학습 인체모델 -> beta" 구조로 구현 가능
- 사전학습 모델은 바로 사용 가능하므로 직접 학습이 즉시 필수는 아님
- 다만 실제 서비스 품질 확보를 위해서는 도메인 적응 파인튜닝이 권장됨
