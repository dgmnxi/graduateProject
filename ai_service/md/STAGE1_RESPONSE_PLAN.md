# Stage 1 대응안: 체형(Beta) 예측 성능 개선 계획

## 0) 이번 졸업프로젝트에서의 목표

현재 Stage 1의 핵심은 단순히 기존 모델을 붙여보는 것이 아니라, **AGORA 720p + 3DPW를 이용해 beta(10-dim)를 직접 회귀하는 자체 모델 파이프라인을 설계하고, 어떤 구조가 가장 안정적으로 작동하는지 비교 검증하는 것**입니다.

### 최종 목표

- beta 10개 차원 각각의 MSE를 **0 ~ 0.25 범위**로 낮추는 것
- 3개 정도의 후보 모델을 학습/추론해보고, 가장 좋은 모델을 최종 선택하는 것
- 학술적으로는 **synthetic-to-real domain adaptation**, **model architecture comparison**, **per-beta error analysis**를 수행하는 졸업프로젝트로 정리하는 것

### 이번 시도가 학술적으로 의미 있는 이유

1. 단순한 모델 연결이 아니라, **데이터셋 혼합 전략(AGORA pretrain + 3DPW finetune)**을 실험한다.
2. 같은 입력(관절/이미지 기반)에서 **모델 구조별 차이(MLP, Transformer, GCN)**가 beta 예측에 어떤 영향을 주는지 비교한다.
3. 전체 평균 MSE만 보지 않고 **beta 차원별 오차**를 분석해, 어떤 체형 요인이 더 어려운지 설명한다.
4. 합성 데이터와 실사 데이터의 분포 차이를 다루므로, 졸업프로젝트 수준에서 충분히 연구 질문이 성립한다.

### 현재 코드 기준에서 중요한 전제

- `ai_service/training` 폴더는 이미 **99차원 관절 입력 → 10차원 beta 회귀** 구조를 전제로 작성되어 있다.
- 즉, 지금 필요한 것은 새로운 서비스 전체가 아니라 **데이터 전처리 + 학습 전략 + 모델 비교 실험 설계**다.
- 학습/추론은 **WSL 환경**에서 수행하고, 결과는 `outputs/checkpoints/`와 `outputs/logs/`에 저장하는 흐름을 유지한다.

## 0-1) 추천 모델 3개

아래 3개를 우선 비교하는 구성이 가장 현실적이다.

### 1. MLP

- 가장 빠른 기준선
- 장점: 학습이 쉽고 디버깅이 빠름
- 역할: 데이터가 실제로 beta 정보를 담는지 빠르게 확인하는 baseline

### 2. Transformer

- 관절 간 전역 관계를 학습하는 모델
- 장점: 전체 관절 패턴을 잡는 데 유리
- 역할: 구조적 상호작용을 잘 잡는지 확인하는 주력 후보

### 3. GCN

- 스켈레톤 연결 구조를 직접 반영하는 모델
- 장점: 관절의 그래프 구조를 이용할 수 있음
- 역할: 인체 골격 구조를 명시적으로 반영했을 때 성능이 오르는지 검증

### 보조 후보

- ResidualMLP는 MLP의 확장형으로 같이 돌려볼 수 있지만, 이번 졸업프로젝트의 "핵심 3개"는 위 세 모델로 두는 것을 권장한다.

## 0-2) 데이터셋 역할 분리

### AGORA 720p

- 역할: **pretrain용 메인 데이터셋**
- 이유: 체형 다양성이 크고 beta supervision이 상대적으로 깔끔함
- 기대 효과: 모델이 체형의 폭을 먼저 배우게 함

### 3DPW

- 역할: **finetune / domain adaptation / final evaluation**
- 이유: 실환경 분포에 가깝고 현재 프로젝트 자산과 잘 맞음
- 기대 효과: AGORA에서 배운 체형 표현을 실사 도메인으로 옮김

### 권장 사용 방식

- pretrain: AGORA
- finetune: 3DPW train
- validation: 3DPW val
- final test: 3DPW test

## 0-4) 학습 전략 원칙

1. 처음부터 큰 모델을 쓰지 말고, **작은 baseline → 중간 모델 → 구조 모델** 순서로 진행한다.
2. 평균 MSE만 보지 말고, **beta별 MSE와 MAE**를 같이 본다.
3. AGORA와 3DPW를 섞을 때는 무조건 합치기보다, **pretrain / finetune 분리**가 먼저다.
4. WSL에서 돌릴 것이므로, **데이터 전처리와 학습 속도**를 먼저 안정화한다.
5. 실패한 실험도 기록해서, 최종 보고서에서 모델 선택 근거로 사용한다.

## 0-6) AGORA로 바꾸는 것만으로는 왜 부족한가

결론부터 말하면, **데이터셋을 AGORA로만 바꾸는 것 자체는 충분한 연구가 아니다**. 지금 실패했던 핵심 원인은 단순히 3DPW가 나빠서가 아니라, **입력 표현이 체형(beta) 정보를 충분히 담지 못했다는 점**이다.

### 이전 시도와 이번 시도의 차이

- 이전 시도: **3DPW 이미지를 MediaPipe에 통과시켜 얻은 관절값 + 얼굴 면적 파생 feature**
- 이번 시도: **전신 이미지/실루엣 중심 입력 + AGORA pretrain + 3DPW finetune**

즉, 이번 실험은 데이터만 갈아끼우는 것이 아니라 다음이 함께 바뀐다.

1. 입력 신호가 관절 좌표에서 전신 이미지/실루엣으로 확장된다.
2. 학습 목표가 단순 회귀가 아니라 synthetic-to-real 전이까지 포함한다.
3. 서비스 목표가 "포즈 추정"이 아니라 "정면 전신 사진에서 체형 beta 추정 후 패션 프롬프트 생성"으로 명확해진다.

### 얼굴 면적이 실패한 이유

- 얼굴 면적은 체형을 직접 설명하는 변수라기보다, 거리/카메라/검출 오차의 영향을 많이 받는 간접 변수였다.
- 그래서 얼굴 feature를 더 추가하는 방식은 **동일한 한계 안에서 feature만 늘리는 반복**에 가깝다.
- 반대로 전신 이미지, 실루엣, bbox, pose heatmap은 체형과 직접 연결될 가능성이 더 크다.

### 3DPW + MediaPipe로 바꿔 적어도 결론이 크게 달라지지 않는 이유

- 3DPW를 MediaPipe에 넣어 얻는 값은 결국 **관절 좌표 기반의 2차 표현**이다.
- 따라서 데이터셋이 3DPW이든 AGORA이든, 입력 표현이 관절 중심이면 beta 회귀의 상한은 크게 바뀌지 않는다.
- 즉, 실패의 핵심은 데이터셋 이름보다 **입력 표현이 체형 정보를 충분히 담지 못한 구조**에 있다.
- 그래서 이번 실험은 데이터셋 교체가 아니라 **입력 표현과 학습 전략의 교체**로 정의해야 한다.

### 그래서 AGORA가 의미 있으려면

AGORA는 단순한 데이터 교체가 아니라, 아래 조건이 같이 있어야 의미가 있다.

- 전신 이미지 기반 backbone을 쓴다.
- AGORA로 체형 다양성을 먼저 학습한다.
- 3DPW로 실사 도메인 적응을 한다.
- 최종 평가는 서비스 입력과 유사한 정면 전신 사진에서 수행한다.

이렇게 해야 "3DPW에서 안 되던 걸 AGORA로 바꿨더니 되는가"가 아니라, **"체형 정보가 더 잘 드러나는 표현으로 바꾸고, 합성 데이터로 먼저 학습한 뒤 실사로 옮기면 beta 회귀가 얼마나 개선되는가"**라는 연구 질문이 된다.

### 졸업프로젝트로서의 의미

이 프로젝트의 학술적 의미는 기존 모델을 단순 재현하는 데 있지 않고, 아래 질문에 답하는 데 있다.

- 어떤 입력 표현이 beta 회귀에 가장 유리한가
- synthetic 데이터가 real 데이터의 성능을 얼마나 끌어올리는가
- beta 차원별로 어떤 체형 정보가 더 어려운가
- 최종 서비스 시나리오에서 가장 안정적인 모델은 무엇인가

## 0-7) 서비스 시나리오에 맞춘 최종 방향

현재 서비스는 다음 흐름이다.

1. 사용자가 **특정 자세로 서 있는 전신 사진**을 입력한다.
2. **우리가 학습한 자체 모델**이 **SMPL beta(10-dim)** 를 회귀한다.
3. beta를 바탕으로 체형 특징을 `마름/보통/비만` 같은 표현으로 정리한다.
4. 체형 특징과 다른 태그를 결합한 프롬프트를 만든다.
5. 그 프롬프트를 Marqo에 넣어 패션 아이템을 검색한다.

따라서 이번 Stage 1의 목표는 이미 존재하는 완성형 SMPL 모델을 그대로 쓰는 것이 아니라, **우리 데이터와 서비스 목표에 맞게 학습한 image-to-beta 회귀 모델**을 만드는 것이다.

### 기존 모델을 쓰는 것과 우리의 연구 범위

- HMR2, PARE, CLIFF 같은 모델은 **비교용 baseline** 또는 빠른 검증용으로는 쓸 수 있다.
- 하지만 졸업프로젝트의 핵심 산출물은 그 모델을 그대로 도입하는 것이 아니라, **우리 데이터에 맞게 학습한 beta 예측기**여야 한다.
- 즉, "SMPL beta를 예측한다"는 말은 **SMPL의 beta 값을 정답으로 두고 우리가 별도의 회귀 모델을 학습한다**는 뜻이다.
- 이 방식이면 학술적으로도 단순 조합이 아니라, 데이터 전이와 모델 구조 비교를 수행한 자체 연구가 된다.

### 그래서 권장하는 실험 우선순위

1. 관절-only는 기준선으로만 둔다.
2. 전신 이미지 기반 모델 3개를 비교한다.
3. AGORA로 pretrain하고 3DPW로 finetune한다.
4. 최종적으로 beta 예측이 안정적인 1개 모델을 고른다.
5. 그 모델을 프롬프트 생성과 Marqo 검색의 앞단에 연결한다.

## 1) 현재 상황 요약

현재 실험 결과 기준으로, `MediaPipe 관절값(99)` + `얼굴 면적 파생 feature`만으로는 SMPL/STAR beta를 신뢰도 있게 예측하기 어렵습니다.

- 관절-only 학습 성능: MSE 0.6 이상
- 얼굴 파생 feature 상관 분석:
  - 최대 절대 Pearson 상관: 약 0.134
  - 단일 feature 선형 회귀 R2: 최대 약 0.018
- 해석:
  - 체형(둘레/부피/지방 분포) 정보가 관절 좌표와 약한 2D/3D landmark만으로 충분히 표현되지 않음

결론적으로, 현재 단계에서는 전략 B(입력 신호 대규모 확장)보다 전략 A(사전학습 모델 활용) + 전략 C(데이터셋 보강) 조합이 현실적입니다.

## 2) 목표 재정의

### 단기 목표 (1~2주)

- 관절-only 기반 beta 예측 경로를 "주 경로"에서 "보조 경로"로 격하
- 사전학습 인체 메쉬/체형 모델을 붙여 baseline 재구축
- 동일 평가셋에서 MSE/RMSE/R2를 재측정

### 중기 목표 (3~6주)

- 데이터셋 품질(체형 다양성, 타깃 신뢰성) 점검 및 보강
- 사전학습 모델 파인튜닝/도메인 적응으로 beta 안정성 확보

## 3) 우선순위 대응 전략

## 전략 B: 자체 모델 개발 (연구적 접근, 성능 최적화) ⭐

**목표**: 자신의 데이터(AGORA/SSP-3D/3DPW)로 다양한 아키텍처를 학습시켜 서비스에 최적화된 모델 구축

### B-1. Transformer 기반 모델

**아키텍처:**
- Backbone: Vision Transformer (ViT) 또는 BERT 변형
- Input: 마스킹된 전신 이미지 (224×224 또는 higher resolution)
- Output: beta (10-dim) + confidence
- Intermediate: image patch embeddings → transformer encoder → regression head

**강점:**
- Self-attention으로 신체 부위 간 관계 학습
- 장거리 의존성(전역 체형 특성) 포착
- 마스킹된 이미지에서 체형 정보 추출에 최적

**예상 성능:**
- MSE ~0.15-0.25 (baseline 관절-only 대비 50% 이상 개선)
- 시간 복잡도: 중상

---

### B-2. MLP 기반 모델

**아키텍처:**
- Backbone: pretrained CNN (ResNet50/EfficientNet) 또는 ViT의 feature extractor
- Intermediate feature: CNN에서 추출한 feature map 또는 flattened vector
- Head: Multi-layer perceptron (3-5 layers)
- Output: beta + confidence

**강점:**
- 매우 빠른 학습
- 파라미터 효율적 (가볍고 빠른 추론)
- Ablation study 용이

**예상 성능:**
- MSE ~0.20-0.30 (적절한 feature extraction이 있으면 경쟁력 있음)
- 시간 복잡도: 낮음

**주의점:**
- feature extraction이 핵심 (backbone 품질 중요)
- spatial 정보 손실 가능성

---

### B-3. GCN (Graph Convolutional Network) 기반 모델

**아키텍처 옵션 1: 이미지 + 스켈레톤 그래프 결합**
- Input A: 마스킹된 이미지
- Input B: SMPL 스켈레톤 그래프 (24개 관절)
- 흐름:
  1. CNN backbone에서 이미지 feature 추출
  2. MediaPipe/pretrained pose detector에서 초기 pose 추정
  3. GCN에서 pose + 신체 그래프 구조를 활용해 beta 예측
- Output: beta + confidence

**아키텍처 옵션 2: 순수 그래프 기반 (pose만 입력)**
- Input: 관절 좌표 (3D landmarks)
- Graph: SMPL skeleton connectivity
- GCN: multi-layer graph convolution + pooling → regression head
- Output: beta

**강점 (옵션 1):**
- 신체 구조를 명시적으로 모델링
- 이미지와 스켈레톤 정보 융합 가능
- 해석 가능성 높음

**강점 (옵션 2):**
- 매우 간단하고 빠름
- 관절-only 방식의 한계를 그래프 구조로 보완

**약점:**
- 그래프 정의 및 초기화 복잡
- 옵션 2는 결국 관절 정보에만 의존 (현재 한계와 유사)

**예상 성능:**
- MSE ~0.18-0.28 (구조 활용 정도에 따라 변동)

---

### B-4. 하이브리드 모델 (특징 수준 융합, 권장) vs 앙상블 (의사결정 수준)

#### B-4-1: 하이브리드 모델 (Feature-level Fusion)

**아키텍처:**

각 모델이 "최종 beta 예측"이 아닌 "중간 표현(intermediate features)"만 생성하고, 이들을 융합한 후 **1개의 회귀 헤드**가 최종 예측

```
마스킹 이미지 → CNN backbone (ResNet50) → multi-scale features
                      ↓
            ┌─────────┼─────────┐
            ↓         ↓         ↓
        Transformer  MLP      GCN
       (encoder)  (dense)   (gcn_layer)
   trans_features mlp_feat gcn_features
            ↓         ↓         ↓
            └─────────┼─────────┘
                      ↓
        concatenate([trans_feat, mlp_feat, gcn_feat])
                      ↓
          Fusion 모듈 (Attention or Weighted Sum)
                      ↓
        Single Regression Head (1개만 존재)
                      ↓
                  beta (10-dim)
```

**특징:**
- 각 아키텍처가 **서로 다른 특징 관점**을 학습
  - Transformer: 전역 구조 및 장거리 의존성
  - MLP: 경량 비선형 변환
  - GCN: 그래프 구조(신체 골격) 활용
- 특징들이 mid-stage에서 **fusion**되므로 상호작용 가능
- End-to-end 학습으로 전체 최적화
- 파라미터 효율: 중간 수준 (각 아키텍처 + fusion head)

**강점:**
- 다양한 모델의 complementary 특징 활용
- 학습이 일관되고 robust
- interpretability: 각 모듈의 기여도 분석 가능

**약점:**
- 설계 및 구현 복잡도 높음
- fusion 방식 선택 중요 (attention vs weighted sum vs concatenate)

**예상 성능:**
- MSE ~0.10-0.18 (최고 성능)

---

#### B-4-2: 앙상블 모델 (Decision-level Ensemble)

각 모델이 **독립적으로 beta를 예측**하고, 최종 단계에서 결과 결합

```
마스킹 이미지
    ├→ Transformer 모델 → beta_trans (10-dim)
    ├→ MLP 모델 → beta_mlp (10-dim)
    └→ GCN 모델 → beta_gcn (10-dim)

          [앙상블 전략]
          ↓
    ┌─────────────────────┐
    ├ Weighted Average    │
    ├ Voting              │
    ├ Stacking            │
    └─────────────────────┘
          ↓
    최종 beta (10-dim)
```

**앙상블 방식:**

1. **Weighted Average (추천)**
   - `beta_final = w1 * beta_trans + w2 * beta_mlp + w3 * beta_gcn`
   - w1+w2+w3 = 1.0
   - 가중치는 validation set 성능 기반 설정

2. **Voting (분류 문제용, beta 회귀에는 부적절)**
   - 회귀 문제이므로 거의 사용 안 함

3. **Stacking**
   - 각 모델의 예측을 입력으로 받는 meta-learner 학습
   - meta-learner가 최종 beta 결정

**특징:**
- 각 모델이 **완전히 독립적**
- 학습/평가가 단순
- 실패 케이스 보완: 한 모델이 틀려도 다른 모델이 보정

**강점:**
- 구현 매우 간단
- 각 모델을 독립적으로 최적화 가능
- 실패 모드(failure mode)에서 견고성 높음

**약점:**
- 특징 수준에서 상호작용 없음
- 파라미터 비효율 (3개 모델 모두 backbone 중복)
- inference 시간이 하이브리드 대비 3배

**예상 성능:**
- MSE ~0.11-0.19 (하이브리드와 유사하지만 구현이 간단)

---

### B-4 최종 비교

| 항목 | 하이브리드 (Feature Fusion) | 앙상블 (Decision Ensemble) |
|------|---------------------------|--------------------------|
| **구조** | 1개 회귀 헤드 | 3개 회귀 헤드 |
| **특징 상호작용** | 있음 | 없음 |

| **구현 시간** | 2-3주 | 1주 |
| **권장 상황** | 최고 성능 필요 | 빠른 구현, 안정성 우선 |

- 약점:
  - 구현 복잡도와 의존성이 HMR보다 다소 높을 수 있음
  - 세팅에 따라 추론 속도가 느려질 수 있음
- 현재 프로젝트 적합도:
  - 높음 (2순위, HMR2 대안/병행)

### A-3. CLIFF

- 한 줄 설명:
  - 바운딩 박스와 사람 중심 정보를 활용해 카메라/크롭 편차를 줄이도록 설계된 SMPL 회귀 모델.
- 입력/출력:
  - 입력: RGB + bbox 관련 정보
  - 출력: pose, beta, camera
- 강점:
  - 크롭/스케일 변화가 큰 환경에서 상대적으로 안정적
  - 기존 검출 파이프라인(MediaPipe bbox/다른 detector)과 결합하기 쉬움
- 약점:
  - bbox 품질에 영향 받음
  - 데이터 전처리 규칙을 엄격히 맞춰야 성능 편차가 줄어듦
- 현재 프로젝트 적합도:
  - 중상 (3순위, 카메라 편차가 큰 경우 유리)

### A-4. PIXIE (보조 후보)

- 한 줄 설명:
  - 얼굴/손/신체를 함께 다루는 고해상도 인체 모델링 계열.
- 입력/출력:
  - 입력: 얼굴/상반신/전신 이미지 (구성에 따라 다름)
  - 출력: 상세 파라미터(얼굴 포함) + body shape 관련 값
- 강점:
  - 얼굴 단서를 적극 활용하는 확장 실험에 적합
  - 상반신 중심 시나리오에서 강점 가능
- 약점:
  - 전체 파이프라인 복잡도 증가
  - 현재 Stage 1 목표(beta 안정 추정) 대비 과한 구성일 수 있음
- 현재 프로젝트 적합도:
  - 중간 (후속 고도화 단계에서 검토)

## 3-2) 전략 C 데이터셋 후보 상세 설명

### C-1. AGORA

- 한 줄 설명:
  - 다양한 체형/포즈/뷰 조건에서 3D 인체 파라미터 학습에 널리 쓰이는 데이터셋.
- 장점:
  - 체형 다양성이 좋아 beta 일반화에 유리
  - 복잡한 장면/포즈 대응력 검증 가능
- 한계:
  - 전처리 비용이 큼
  - 현재 파이프라인과 라벨 정합 작업 필요

### C-2. SSP-3D

- 한 줄 설명:
  - shape 정확도 평가 목적으로 자주 인용되는 체형 중심 벤치마크 성격 데이터셋.
- 장점:
  - 체형 평가 지표 비교에 유용
  - beta 추정 성능 검증에 직접적
- 한계:
  - 학습용 대규모 데이터로 단독 사용하기엔 제약 가능

### C-3. 3DPW (기존 유지)

- 한 줄 설명:
  - 실환경 포즈/동작 데이터로 널리 사용되는 기본 축 데이터셋.
- 장점:
  - 현재 자산과 연결이 쉬움
  - 시퀀스 기반으로 추론 안정성 테스트 가능
- 한계:
  - 체형 supervision 강도만 보면 shape 특화셋보다 약할 수 있음

### C-4. 혼합 학습 권장안

- 권장 구성:
  - pretrain: AGORA + (가능하면 SSP-3D 일부)
  - finetune: 현재 3DPW/서비스 유사 도메인 데이터
- 기대 효과:
  - 체형 표현력 + 도메인 적응 균형 확보

- 관절-only 대비 체형 관련 신호 확보 가능
- beta 축별 설명력 상승 여지 큼

- 특정 포즈/카메라 조건 편향 완화 가능

- 학습 데이터와 서비스 환경(카메라, 의상, 조명) 차이가 크면 성능 저하

- 고성능 모델은 추론비용이 커서 서비스 적용 시 최적화 필요
