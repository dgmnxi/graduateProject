# Stage 1 대응안: 체형(Beta) 예측 성능 개선 계획

## 1) 현재 상황 요약

현재 실험 결과 기준으로, `MediaPipe 관절값(99)` + `얼굴 면적 파생 feature`만으로는 SMPL/STAR beta를 신뢰도 있게 예측하기 어렵습니다.

- 관절-only 학습 성능: MSE 0.6 이상
- 얼굴 파생 feature 상관 분석:
  - 최대 절대 Pearson 상관: 약 0.134
  - 단일 feature 선형 회귀 R2: 최대 약 0.018
- 해석:
  - 체형(둘레/부피/지방 분포) 정보가 관절 좌표와 약한 2D/3D landmark만으로 충분히 표현되지 않음

결론적으로, 현재 단계에서는 전략 B(입력 신호 대규모 확장)보다 전략 A(사전학습 모델 활용) + 전략 C(데이터셋 보강) 조합이 현실적입니다.

---

## 2) 목표 재정의

### 단기 목표 (1~2주)

- 관절-only 기반 beta 예측 경로를 "주 경로"에서 "보조 경로"로 격하
- 사전학습 인체 메쉬/체형 모델을 붙여 baseline 재구축
- 동일 평가셋에서 MSE/RMSE/R2를 재측정

### 중기 목표 (3~6주)

- 데이터셋 품질(체형 다양성, 타깃 신뢰성) 점검 및 보강
- 사전학습 모델 파인튜닝/도메인 적응으로 beta 안정성 확보

---

## 3) 우선순위 대응 전략

## 전략 A: 기존 사전학습 모델 활용 (최우선)

후보:
- HMR 계열(HMR2)
- PARE
- CLIFF

실행 방식:
1. 우선 추론-only로 beta 결과 생성
2. 현재 파이프라인 평가 코드에 연결
3. 필요 시 마지막 회귀 헤드만 파인튜닝

기대 효과:
- 초기 구현 속도가 빠름
- 관절-only 한계를 즉시 우회 가능

주의점:
- 카메라 파라미터/스케일 보정 없으면 beta 오차가 커질 수 있음

## 전략 C: 데이터셋 재선정/보강

권장 방향:
- shape supervision 신뢰도가 높은 데이터 추가
- 가능한 경우 AGORA, SSP-3D 등 체형 라벨이 강한 셋 포함
- 기존 3DPW 계열은 보조/도메인 적응 용도로 병행

기대 효과:
- 타깃 노이즈 감소
- beta 축별 편향 완화

주의점:
- 라이선스, 전처리 비용, 도메인 차이 고려 필요

---

## 3-1) 전략 A 후보 모델 상세 설명

현재 프로젝트(체형 beta 예측)에서 우선 검토할 후보는 아래 4개입니다.

### A-1. HMR2

- 한 줄 설명:
  - 단일 이미지에서 SMPL 파라미터(포즈/shape/camera)를 추정하는 HMR 계열의 최신 개선형.
- 입력/출력:
  - 입력: 전신 RGB 이미지
  - 출력: pose(회전), beta(shape), camera
- 강점:
  - 공개 구현과 재현 사례가 많아 baseline 구축이 빠름
  - 체형/포즈를 동시에 추정하므로 현재 목표와 직접 정렬
  - 추론-only로 먼저 붙이기 쉬움
- 약점:
  - 카메라/거리 편차에 민감
  - 데이터 도메인(의상/촬영각) 차이가 크면 beta 불안정 가능
- 현재 프로젝트 적합도:
  - 매우 높음 (1순위)

### A-2. PARE

- 한 줄 설명:
  - 가려짐(occlusion) 상황에서 부위 기반 attention으로 SMPL 추정 성능을 높인 모델.
- 입력/출력:
  - 입력: RGB 이미지
  - 출력: pose, beta, camera(구현체에 따라 형태 차이)
- 강점:
  - 의상/가림이 있는 실환경 이미지에서 비교적 견고
  - 체형 회귀에서 실패 케이스(부분 가림) 완화에 유리
- 약점:
  - 구현 복잡도와 의존성이 HMR보다 다소 높을 수 있음
  - 세팅에 따라 추론 속도가 느려질 수 있음
- 현재 프로젝트 적합도:
  - 높음 (2순위, HMR2 대안/병행)

### A-3. CLIFF

- 한 줄 설명:
  - 바운딩 박스와 사람 중심 정보를 활용해 카메라/크롭 편차를 줄이도록 설계된 SMPL 회귀 모델.
- 입력/출력:
  - 입력: RGB + bbox 관련 정보
  - 출력: pose, beta, camera
- 강점:
  - 크롭/스케일 변화가 큰 환경에서 상대적으로 안정적
  - 기존 검출 파이프라인(MediaPipe bbox/다른 detector)과 결합하기 쉬움
- 약점:
  - bbox 품질에 영향 받음
  - 데이터 전처리 규칙을 엄격히 맞춰야 성능 편차가 줄어듦
- 현재 프로젝트 적합도:
  - 중상 (3순위, 카메라 편차가 큰 경우 유리)

### A-4. PIXIE (보조 후보)

- 한 줄 설명:
  - 얼굴/손/신체를 함께 다루는 고해상도 인체 모델링 계열.
- 입력/출력:
  - 입력: 얼굴/상반신/전신 이미지 (구성에 따라 다름)
  - 출력: 상세 파라미터(얼굴 포함) + body shape 관련 값
- 강점:
  - 얼굴 단서를 적극 활용하는 확장 실험에 적합
  - 상반신 중심 시나리오에서 강점 가능
- 약점:
  - 전체 파이프라인 복잡도 증가
  - 현재 Stage 1 목표(beta 안정 추정) 대비 과한 구성일 수 있음
- 현재 프로젝트 적합도:
  - 중간 (후속 고도화 단계에서 검토)

---

## 3-2) 전략 C 데이터셋 후보 상세 설명

### C-1. AGORA

- 한 줄 설명:
  - 다양한 체형/포즈/뷰 조건에서 3D 인체 파라미터 학습에 널리 쓰이는 데이터셋.
- 장점:
  - 체형 다양성이 좋아 beta 일반화에 유리
  - 복잡한 장면/포즈 대응력 검증 가능
- 한계:
  - 전처리 비용이 큼
  - 현재 파이프라인과 라벨 정합 작업 필요

### C-2. SSP-3D

- 한 줄 설명:
  - shape 정확도 평가 목적으로 자주 인용되는 체형 중심 벤치마크 성격 데이터셋.
- 장점:
  - 체형 평가 지표 비교에 유용
  - beta 추정 성능 검증에 직접적
- 한계:
  - 학습용 대규모 데이터로 단독 사용하기엔 제약 가능

### C-3. 3DPW (기존 유지)

- 한 줄 설명:
  - 실환경 포즈/동작 데이터로 널리 사용되는 기본 축 데이터셋.
- 장점:
  - 현재 자산과 연결이 쉬움
  - 시퀀스 기반으로 추론 안정성 테스트 가능
- 한계:
  - 체형 supervision 강도만 보면 shape 특화셋보다 약할 수 있음

### C-4. 혼합 학습 권장안

- 권장 구성:
  - pretrain: AGORA + (가능하면 SSP-3D 일부)
  - finetune: 현재 3DPW/서비스 유사 도메인 데이터
- 기대 효과:
  - 체형 표현력 + 도메인 적응 균형 확보

---

## 3-3) 최종 선택 가이드 (전략 A/C 중심)

### 1단계 (가장 빠른 실행)

- HMR2를 먼저 붙여 추론-only baseline 구축
- 동일 테스트셋에서 기존 모델 대비 지표 비교

### 2단계 (안정화)

- PARE 또는 CLIFF를 추가 비교
- 실패 케이스(가림/거리/크롭)에 강한 모델 선택

### 3단계 (성능 상향)

- 데이터셋 혼합 학습(AGORA/SSP-3D + 3DPW)
- 필요한 범위에서 회귀 헤드 또는 전체 파인튜닝

## 4) 실행 체크리스트

## 4-1. 즉시 실행 (이번 주)

- [ ] beta 타깃 무결성 검사 추가
- [ ] 사전학습 모델 1종(HMR2 또는 PARE) 선정
- [ ] 추론-only 파이프라인 연결
- [ ] 기존 테스트셋에서 동일 지표(MSE/MAE/RMSE/R2) 비교
- [ ] 결과를 outputs/visual에 리포트 저장

## 4-2. 단기 개선 (1~2주)

- [ ] 후보 모델 2종 이상(HMR2 + PARE/CLIFF) 성능 비교
- [ ] 공통 전처리 규격 고정 (bbox, resize, normalization)
- [ ] beta별 성능 편차 분석(어떤 beta가 특히 어려운지)
- [ ] 실패 케이스 분류(가림/거리/크롭 오차)

## 4-3. 중기 개선 (3~6주)

- [ ] 데이터셋 재선정 또는 혼합 학습 구성
- [ ] 카메라 거리/스케일 보정 feature 반영
- [ ] 에러 케이스 클러스터링(포즈/거리/가림/의상 두께)
- [ ] 최종 모델 경량화 및 추론 속도 검증

---

## 5) 가능성과 한계

## 가능성

1. 사전학습 모델 활용 시
- 초기 성능을 빠르게 끌어올릴 가능성이 높음
- end-to-end 재학습 없이도 baseline 대체 가능

2. 이미지/실루엣 입력 확장 시
- 관절-only 대비 체형 관련 신호 확보 가능
- beta 축별 설명력 상승 여지 큼

3. 데이터셋 보강 시
- 일반화 성능 개선 가능
- 특정 포즈/카메라 조건 편향 완화 가능

## 한계

1. 단안 RGB의 본질적 모호성
- 동일 실루엣에서도 실제 체형이 다를 수 있음
- 깊이/거리 정보 부재로 절대 체형 복원 한계

2. 라벨 신뢰도 이슈
- beta 타깃 생성 과정이 불안정하면 모델 상한이 낮아짐

3. 도메인 갭
- 학습 데이터와 서비스 환경(카메라, 의상, 조명) 차이가 크면 성능 저하

4. 실시간 제약
- 고성능 모델은 추론비용이 커서 서비스 적용 시 최적화 필요

---

## 6) 의사결정 기준 (Go / No-Go)

다음 조건을 만족하면 관절-only 경로를 종료하고 이미지 중심 경로로 확정합니다.

- [ ] 얼굴/관절 파생 feature에서 |corr| >= 0.2를 안정적으로 확보하지 못함
- [ ] 단일/단순 모델에서 beta 회귀 R2가 대부분 0.1 미만
- [ ] 사전학습 모델 baseline이 현행 대비 명확히 우수함

권장 임계값(초기):
- 관절-only MSE 개선폭이 15% 미만이면 투자 중단
- 이미지 기반 baseline이 MSE 30% 이상 개선하면 전환 확정

---

## 7) 제안 결론

현재 단계에서 가장 현실적인 해법은 다음 순서입니다.

1. 기존 사전학습 인체 모델을 먼저 붙여 baseline 교체
2. PARE/CLIFF를 대조군으로 병행 평가
3. 관절값은 보조 입력으로 유지
4. 데이터셋을 체형 supervision 중심(AGORA/SSP-3D 보강)으로 재구성

즉, "관절-only 체형 추정"은 연구 보조 트랙으로 유지하고,
"이미지 기반 + 사전학습 모델"을 제품 경로로 채택하는 것을 권장합니다.
